# Content Strategy Workflow Instructions

## Input Context

Read these BMAD configuration files for context:

1. `{project-root}/instructions/01-bmad-config.md` — Project aliases and settings

## Task Overview

Develop a comprehensive content strategy using topic clusters, pillar pages, and editorial calendars to drive organic growth.

## Content Coverage

### Content Audit Methodology
- **Inventory existing content** — URLs, formats, performance metrics
- **Gap analysis** — Missing topics, outdated content, thin content
- **Performance benchmarking** — Top-performing vs underperforming pieces
- **Keyword opportunity mapping** — Search volume, competition, intent

### Topic Cluster and Pillar Strategy
- **Pillar content identification** — Core topics aligned with product value
- **Subtopic cluster mapping** — Supporting articles linking to pillars
- **Content silo structure** — Logical grouping for SEO and user experience
- **Internal linking framework** — Hub-and-spoke navigation patterns

### Editorial Calendar Framework
- **Publishing frequency** — Weekly, bi-weekly, monthly cadence
- **Content types and formats** — Blogs, guides, case studies, videos
- **Seasonal and event planning** — Product launches, holidays, campaigns
- **Resource allocation** — Writer assignments, review cycles, deadlines

### Content Governance and Workflow
- **Approval process** — Draft → Review → Edit → Publish
- **Quality standards** — E-E-A-T compliance, readability, formatting
- **Version control and storage** — Where content lives, who owns updates
- **Performance measurement** — KPIs, reporting, iteration cycles

## Workflow Execution

1. Read BMAD config and product brief for context
2. Conduct content audit with methodology above
3. Map topic clusters and pillar content strategy
4. Design editorial calendar framework
5. Define content governance and workflow

## Final Output

Write results to `{project-root}/_bmad-output/content-strategy-{date}.md` containing:

### Content Strategy Section
- **Topic cluster map** — Pillar pages with subtopic links
- **Content types and formats** — Recommended for each cluster
- **Publishing frequency and cadence** — Weekly/bi-weekly/monthly
- **Content workflow and approval process** — Draft to publish steps
